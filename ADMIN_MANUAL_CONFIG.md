# Admin Portal — Manual Config Guide

## Setup after cloning / first deploy

### 1. Create `website/admin/config.js`

Copy `website/admin/config.example.js` to `website/admin/config.js` and fill in the values:

```js
window.ADMIN_CONFIG = {
  SUPABASE_URL:      'https://nssmpthalivhfbdernye.supabase.co',
  SUPABASE_ANON_KEY: '<your-supabase-anon-key>',
  STORAGE_BUCKET:    'portocoaststays-media',
};
```

> `config.js` is git-ignored. **Never commit credentials.**
> On Netlify the file is auto-generated by `scripts/generate-admin-config.js` during the build.

---

### 2. Set Netlify environment variables

In **Netlify > Site settings > Environment variables**, add:

| Variable | Value |
|---|---|
| `SUPABASE_URL` | `https://nssmpthalivhfbdernye.supabase.co` |
| `SUPABASE_ANON_KEY` | `<anon public key from Supabase Dashboard>` |
| `NETLIFY_BUILD_HOOK_URL` | `<build hook URL from Netlify>` |
| `STORAGE_BUCKET` | `portocoaststays-media` *(optional — this is the default)* |

**Getting the build hook URL:**
- Netlify > Site > Site settings > Build & deploy > Build hooks > Add build hook → name it `admin-trigger`

---

### 3. Run Supabase migrations

Run the SQL files in order in the **Supabase SQL Editor**:

```
001_profiles_and_trigger.sql
002_core_rpcs.sql
003_admin_rpcs.sql
004_posts.sql
005_media_assets.sql
006_storage_bucket.sql
007_properties.sql
008_task_templates.sql
008b_task_templates_seed.sql
009_task_tracking.sql
```

---

### 4. Create the first admin user

1. Go to `https://portocoaststays.com/admin/register` and register with your email.
2. Confirm your email via the Supabase confirmation link.
3. In **Supabase > Table editor > profiles**, manually set your row:
   - `role` → `admin`
   - `client` → `portocoaststays`
4. You can now log in and promote other users via the **Utilizadores** page.

---

### 5. Create the Storage bucket

If migration `006_storage_bucket.sql` fails on the INSERT into `storage.buckets`:
1. Go to **Supabase > Storage > New bucket**
2. Name: `portocoaststays-media`
3. Toggle **Public bucket**: ✅
4. Then run only the `CREATE POLICY` lines from `006_storage_bucket.sql`.

## User roles

| Role | `client` | Can do |
|---|---|---|
| `admin` | `portocoaststays` | Everything — all properties, all tasks, all blog posts, media, users |
| `user` | `portocoaststays` | Own properties, own task statuses, shared comments (no internal) |
| Pending | anything else | Cannot log in; admin must run **Ativar cliente** on the Users page |

## Triggering a public site rebuild

After publishing or updating posts, use the **🌐 Publicar Site** button in the admin portal.
This calls `/.netlify/functions/trigger-build`, which fires the Netlify build hook.
The public site regenerates in ~1 minute.
